Components

The following components or web elements are currently functional in the CVA interface. Classes for each component and character style are assigned in the Word document then rendered into markup and Angular functions from the parser. Each component and character style has default CSS in the base template but can be customized with override styles.

Headers

There are three types of headers in the CVA.

A-Head

A-Heads are <h1> header tags that contain the chapter header and triggers a chapter break.

Docx: Assign the # A Head class to the title of each new chapter.

Default A-Head font sizes are responsive according to the breakpoints in following mixin:

@mixin scaled-header-font($large:52px,$medium:46px,$small:36px) { @media only screen and (min-width:$small-screen) { font-size: $large; max-width: 760px; } @media only screen and (min-width:$mobile-large-screen) and (max-width:$small-screen) { font-size: $medium; max-width: 650px; } @media only screen and (max-width:$mobile-large-screen) { font-size: $small; max-width: 520px; } }

Default A-Head CSS

.chapter-header { @include scaled-header-font; width: 100%; margin: 2rem 0; padding: 0; text-transform: uppercase; line-height: $line-height-medium; overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }

B-Head

B-Heads are <h3> header tags that contain the page header and triggers a page break within a chapter of a manual.

Docx: Assign the # B Head class to the title at the beginning of each new page.

Default B-Head CSS:

.bhead { font-size: $font-size-medium; #1.5rem/24px }

Body Head

Body Heads are also <h3> header tags, identical to B-Heads but do not trigger a chapter or page break.

Docx: Assign the # Body Head class to any line of text to create a header anywhere on a page.

Default Body Head CSS:

.body-head { font-size: $font-size-medium; }

For custom backgrounds, add a background-image rule to the .chapter-header class in the CSS overrides.

Content

CVA content is laid out in the following ways:

Body Normal

Static manual content is rendered as <p> paragraph block elements.

.body-normal { font-size: $font-size-normal; font-weight: normal; padding: 0; margin: 0 0 1rem 0; }

Assign the # Body Normal class to any static content in the Word document.

Bulleted List

The CPT renders bulleted lists as <ul> list elements. The default list-style can be changed or hidden with CSS with the list-style-type attribute.

Docx: Assign the # Bulleted List class to the contents of the desired list.

External Link

The CPT renders external links as <a> elements. A class of .body-hyperlink is assigned to the <a> tag and the href location is defined in the CPT.

Docx: Enter the hyperlink copy. Then on the same line, with no space, enter the link URL inside [ square brackets ] and omit the http:// prefix. Assign the # External Link class to the entire link copy. Example: Click Me[www.example.com]

Control Items

Control Items are PlaystationTM specific SVG icons representing each controller button. They are mainly used to illustrate game controls and move instructions in manuals.

Modals

Modal Images

Modal Images are image thumbnails with a trigger action to expand the image to full screen. Upload images in the CPT and make sure the image files are named with a prefix: modal_image_[filename].jpg In the Word document, insert the file name where a modal image is needed, then assign the # Modal Image class to the line containing the file name.
